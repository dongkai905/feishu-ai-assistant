## 部署步骤

### 步骤1: Docker环境准备
```bash
# 等待Docker安装完成
# 验证安装
docker --version
docker-compose --version
```

### 步骤2: 构建核心服务
```bash
# 只构建核心服务
docker-compose build api postgres redis nginx
```

### 步骤3: 启动核心服务
```bash
# 启动最小服务集
docker-compose up -d api postgres redis nginx
```

### 步骤4: 验证部署
```bash
# 检查服务状态
docker-compose ps

# 测试API
curl http://localhost:8080/health
curl http://localhost:8080/version
```

### 步骤5: 逐步扩展
```bash
# 添加监控
docker-compose up -d prometheus grafana

# 添加异步任务
docker-compose up -d celery
```

## 应急方案

### 如果Docker安装失败
1. **使用开发服务器**: 继续使用 `python3 src/main.py`
2. **云部署**: 考虑部署到云服务 (AWS, 阿里云)
3. **虚拟机**: 使用虚拟机运行Docker

### 如果API修复需要时间
1. **部署现有功能**: 先部署工作正常的端点
2. **逐步修复**: 在部署后修复问题
3. **版本控制**: 使用Git管理不同版本
